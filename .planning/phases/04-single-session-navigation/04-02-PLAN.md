    Replace the placeholder Player.tsx with the full slideshow-style navigation implementation.

    **Player component structure:**

    1. **Data sourcing:** Import `useSessionStore` and read `activeSession`. Import `useNavigationStore` and read `steps`, `currentStepIndex`, `expandedSteps`, and all action functions. Call `useKeyboardNavigation()` to activate keyboard bindings.

    2. **Initialization:** In a `useEffect` watching `activeSession`, call `navigationStore.initializeSteps(activeSession.messages)` when activeSession is not null. Call `navigationStore.reset()` on cleanup (unmount).

    3. **Empty state:** If `activeSession` is null or `steps.length === 0`, show a centered message: "No session loaded. Select a session in Builder mode to preview it here." with a link or instruction to go to `/builder`. This is the state when user navigates directly to `/player` without selecting a session.

    4. **Fade transition:** Use a `useState<boolean>` called `visible` (starts true). When `currentStepIndex` changes (track with `useEffect`), set `visible` to false, then after a ~75ms timeout set it to true. This creates a brief opacity dip (75ms out + 75ms in = 150ms total). Apply `opacity: visible ? 1 : 0` with `transition: opacity 75ms ease-in-out` to the step container.

    5. **Tool use map:** Build the tool use map from `activeSession.messages` (same `buildToolUseMap` pattern used in MessageList -- can inline it or import). This map is needed by MessageBubble for rejection display. Compute once with `useMemo` depending on `activeSession`.

    6. **Layout:** Full viewport height (minus titlebar). The step content area takes all available space. NavigationControls overlay on left/right edges. ProgressIndicator is fixed bottom-right.

    7. **Render current step:**
       ```
       const currentStep = steps[currentStepIndex]
       const expanded = expandedSteps[currentStepIndex] ?? { user: false, assistant: false }
       ```
       Render `<StepView step={currentStep} expandedState={expanded} onToggleExpand={(role) => toggleExpand(currentStepIndex, role)} toolUseMap={toolUseMap} />` inside the fading container.

    8. **Navigation controls:**
       ```
       <NavigationControls
         canGoBack={currentStepIndex > 0}
         canGoForward={currentStepIndex < steps.length - 1}
         onBack={prevStep}
         onForward={nextStep}
       />
       ```

    9. **Progress indicator:**
       ```
       <ProgressIndicator currentStep={currentStepIndex} totalSteps={steps.length} />
       ```

    10. **Scroll reset:** In the useEffect that watches `currentStepIndex`, also scroll the step container to top (ref the container div, call `scrollTo(0, 0)`). This ensures that when navigating to a step that was previously scrolled down (if expanded), the view starts at the top.

    Per user decisions honored:
    - Slideshow-style: only current step rendered (not a list)
    - Responses start collapsed (expandedSteps defaults to empty = all collapsed)
    - Expand/collapse remembered (stored in Zustand navigationStore)
    - Forward/backward move between pairs regardless of collapse state
    - Presentation feel: 20px font via presentation-mode class on StepView
