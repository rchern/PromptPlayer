**1. Create `.github/workflows/ci.yml` (PR checks):**

```yaml
name: CI

on:
  pull_request:
    branches: [main]

jobs:
  check:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm
      - run: npm ci
      - run: npm run lint
      - run: npm run typecheck
      - run: npm run build
```

Per user decision: lint + build steps in CI, no test suite. The `typecheck` step runs both node and web tsconfigs. The `build` step runs `electron-vite build` to verify the app compiles.

**2. Create `.github/workflows/release.yml` (tag-triggered release):**

```yaml
name: Release

on:
  push:
    tags:
      - 'v*.*.*'

jobs:
  build:
    runs-on: windows-latest
    permissions:
      contents: write
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm
      - run: npm ci
      - run: npm run build
      - name: Build Windows installer
        env:
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        run: npx electron-builder --win --publish always
```

The `--publish always` combined with `releaseType: draft` in electron-builder.yml creates a draft GitHub Release and uploads the installer .exe + latest.yml metadata. The `GH_TOKEN` from `secrets.GITHUB_TOKEN` is automatically available in GitHub Actions.

Per user decision: draft releases so artifacts can be reviewed before publishing. Auto-updater (electron-updater) only detects published releases, not drafts -- this is the desired behavior.
