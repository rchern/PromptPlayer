    A summary stats header displayed at the top of the preview panel when a session is selected. Per locked decision: "Summary header includes: message/step count, session duration, project path."

    Props:
    ```typescript
    interface SessionPreviewHeaderProps {
      session: StitchedSession
      metadata: SessionMetadata | null
    }
    ```

    Implementation:
    1. Compute narrative message count: `filterVisibleMessages(session.messages, false).length`
    2. Compute step count: `buildNavigationSteps(filterVisibleMessages(session.messages, false)).length` -- import both from `../../utils/messageFiltering`
    3. Compute duration: If metadata has firstTimestamp and lastTimestamp, use `formatSessionDuration(metadata)` from `../../types/pipeline`. Otherwise "Unknown".
    4. Get project path from `metadata?.projectFolder ?? 'Unknown'`
    5. Get command snippet: If `metadata?.firstUserMessage` starts with `/`, show it as "Command" stat. Per discretion: yes, include key commands in preview summary.

    Layout: A horizontal stat grid using CSS Grid with `repeat(auto-fit, minmax(120px, 1fr))`. Each stat is a small card with:
    - Label (uppercase, small, muted text)
    - Value (larger, primary text)

    Stats to show: Messages | Steps | Duration | Project | Command (if applicable)

    Styling: Use `var(--color-bg-tertiary)` background, `var(--color-border)` bottom border, `var(--radius-md)` border radius. Match the existing StatCard pattern from Builder.tsx but make it horizontal and more compact.

    Wrap the computation in `useMemo` to avoid recalculating on every render:
    ```typescript
    const stats = useMemo(() => {
      const visible = filterVisibleMessages(session.messages, false)
      const steps = buildNavigationSteps(visible)
      return { messageCount: visible.length, stepCount: steps.length }
    }, [session.messages])
    ```
