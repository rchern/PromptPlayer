   - Import `ToolCategoryConfig` from the presentation types
   - This function coexists with the existing `filterVisibleMessages` (do NOT modify or remove it -- it's used by the Player)
   - Signature: `export function filterWithToolSettings(messages: ParsedMessage[], toolVisibility: ToolCategoryConfig[]): ParsedMessage[]`
   - Logic:
     a. Build a flat `Map<string, boolean>` lookup from tool name to visibility. For each category, iterate tools; per-tool override in `toolOverrides` takes precedence over `category.visible`.
     b. Handle the "Other" category (empty tools array): store its `visible` value as a default for unknown tools.
     c. Filter messages using the same rules as `filterVisibleMessages` but with the map instead of a boolean:
        - Skip meta messages (`msg.isMeta`)
        - Skip empty-after-cleaning user messages (`isEmptyAfterCleaning(msg)`)
        - toolVisibility null: always show (pure text)
        - toolVisibility 'narrative': check if specific tool is hidden in the map. If tool IS hidden but message has meaningful text, still show (mixed-content exception)
        - toolVisibility 'unknown': use the "Other" category's visible setting. If hidden but has text, still show.
        - toolVisibility 'plumbing': check if specific tool is enabled in the map. If enabled, show. If not but has meaningful text, still show (mixed-content exception, same as existing).
        - Default: hide
