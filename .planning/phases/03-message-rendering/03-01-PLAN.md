  <action>
1. Create `src/renderer/src/components/message/CodeBlock.tsx`:
   - Props: `{ language: string | undefined; children: React.ReactNode }`
   - Renders a `.code-block-wrapper` div containing:
     - A `.code-block-lang` span showing the language name (if language is defined and not empty). Map common language identifiers to display names: "typescript"/"ts" -> "TypeScript", "javascript"/"js" -> "JavaScript", "python"/"py" -> "Python", "bash"/"shell"/"sh" -> "Shell", "json" -> "JSON", "css" -> "CSS", "html" -> "HTML", "csharp"/"cs" -> "C#", "sql" -> "SQL", etc. Fallback: capitalize the raw language string.
     - A `<pre>` element that passes through children (which will be the shiki-processed `<code>` element)
   - Use a ref + useEffect to detect if the `<pre>` element has overflow (scrollHeight > clientHeight). If so, add class `has-overflow` to the wrapper div. This activates the CSS fade hint.
   - Use React.JSX.Element return type (not JSX.Element) per project convention.

2. Create `src/renderer/src/components/message/MarkdownRenderer.tsx`:
   - Props: `{ content: string }`
   - Import `Markdown` from `react-markdown`, `remarkGfm` from `remark-gfm`, `rehypeShiki` from `@shikijs/rehype`
   - Configure shiki options as a module-level constant (NOT inside the component to avoid re-creation):
     ```typescript
     const SHIKI_OPTIONS = {
       themes: {
         light: 'github-light',
         dark: 'github-dark',
       },
     }
     ```
   - Render: `<div className="markdown-body message-view">` wrapping `<Markdown>` with:
     - `remarkPlugins={[remarkGfm]}`
     - `rehypePlugins={[[rehypeShiki, SHIKI_OPTIONS]]}` (note: array-in-array for plugin with options)
     - `components` prop overriding:
       - `pre`: Extract the language from the child `<code>` element's className (react-markdown sets `className="language-xxx"` on code elements inside pre). Pass to CodeBlock component. Pattern: check `props.children` is a React element with className matching `/language-(\S+)/`, extract capture group.
       - `table`: Wrap in `<div className="table-wrapper">` for horizontal scroll on wide tables.
     - Note: Do NOT override the `code` component. @shikijs/rehype processes code blocks at the AST level and replaces the `<code>` element contents. Custom `code` component would interfere.
   - Use React.JSX.Element return type.
   - IMPORTANT: react-markdown v10 removed the `className` prop on `<Markdown>`. That's why we wrap in our own div.
  </action>
