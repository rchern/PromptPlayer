1. Create `src/renderer/src/components/message/ThinkingBlock.tsx`:
   - Props: `{ content: string }`
   - Renders a collapsed toggle showing "Thinking..." in a dimmed, small style
   - Use React useState for `isExpanded` (default: false)
   - Collapsed state: A single line with a chevron icon (use `ChevronRight` from lucide-react, rotated 90deg when expanded) + italic dimmed text "Thinking..." + small character count hint like "(1,234 chars)"
   - Expanded state: Shows the full thinking text in a dimmed, slightly smaller font (0.9em) with a light left border (similar to blockquote styling). The text is plain -- do NOT run it through MarkdownRenderer (thinking blocks are internal reasoning, not formatted output).
   - Styling: Use CSS variables from theme.css. The collapsed row should have padding var(--space-2) var(--space-3), cursor: pointer. Color: var(--color-text-muted). 2. Create `src/renderer/src/components/message/ToolCallBlock.tsx`:
   - Props: `{ name: string; input: Record<string, unknown>; toolUseId: string }`
   - Renders a compact card showing the tool name and a brief input summary
   - Tool name: displayed in monospace font (var(--font-mono)), accent color (var(--color-accent)), font-weight 600, font-size var(--text-sm)
   - Input summary: create a `summarizeToolInput(name, input)` helper function:
     - If name is 'AskUserQuestion' and input.question is string: show question (truncated to 100 chars)
     - If input has 'description' string: show it (truncated to 100 chars)
     - If input has 'command' string (e.g., Bash tool): show it (truncated to 80 chars)
     - If input has 'file_path' or 'path' string: show it
     - Fallback: show first string value from input (truncated to 80 chars)
     - If no string values: return empty string
   - Summary text: font-size var(--text-sm), color var(--color-text-muted), monospace, single line with text-overflow: ellipsis
   - Container: background var(--color-bg-tertiary), border 1px solid var(--color-border), border-radius var(--radius-md), padding var(--space-2) var(--space-3), margin var(--space-2) 0
   - Also export a `ToolResultBlock` component (same file):
     - Props: `{ content: string; isError: boolean }`
     - If content is empty, return null
     - Truncate content to first 200 characters (or first 3 lines, whichever is shorter)
     - Display in a subdued style: font-size var(--text-sm), color var(--color-text-secondary), monospace, padding var(--space-2), max-height 4em, overflow hidden
     - If isError: show with a subtle red tint (color: #ef4444 at 80% opacity)

3. Create `src/renderer/src/components/message/ContentBlockRenderer.tsx`:
   - Props: `{ block: ContentBlock; toolVisibility: ToolVisibility | null; showPlumbing: boolean }`
   - Import types from `../../types/pipeline`
   - Import MarkdownRenderer from `./MarkdownRenderer`
   - Import ThinkingBlock from `./ThinkingBlock`
   - Import ToolCallBlock, ToolResultBlock from `./ToolCallBlock`
   - Switch on `block.type`:
     - `'text'`: Render `<MarkdownRenderer content={block.text} />`
     - `'thinking'`: Render `<ThinkingBlock content={block.thinking} />`
     - `'tool_use'`:
       - If `toolVisibility === 'plumbing' && !showPlumbing`: return null (hidden)
       - Otherwise: Render `<ToolCallBlock name={block.name} input={block.input} toolUseId={block.id} />`
     - `'tool_result'`:
       - If `toolVisibility === 'plumbing' && !showPlumbing`: return null (hidden)
       - Otherwise: Render `<ToolResultBlock content={block.content} isError={block.is_error} />`
     - `'image'`: return null (images not rendered in v1)
     - default: return null
   - Use React.JSX.Element | null return type
