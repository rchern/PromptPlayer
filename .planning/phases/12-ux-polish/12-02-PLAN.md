1. Add `splitToNewSection: (sectionId: string, sessionId: string) => void` to the PresentationState interface
2. Implement the action:
   - Get active presentation
   - Find the source section and the session ref within it
   - Remove the session ref from the source section
   - Create a new PresentationSection with:
     - `id: crypto.randomUUID()`
     - `name: sessionRef.displayName` (name after the session being split out)
     - `sessionRefs: [sessionRef]` (just the one session)
   - Insert the new section immediately AFTER the source section's position in the sections array
   - If the source section is now empty (no remaining sessionRefs), remove it from the array
   - Set `updatedAt: Date.now()`
   - Call `persistPresentation(updated)` to save

This mirrors the research code example in 12-RESEARCH.md (Pattern 8).
