**1. Add Task tool dispatch to ToolCallBlock:**

Import the three new components:
```typescript
import { TaskCreateBlock } from './TaskCreateBlock'
import { TaskUpdateBlock } from './TaskUpdateBlock'
import { TaskListBlock } from './TaskListBlock'
```

Add dispatch branches after the existing AskUserQuestion dispatch (added in Plan 01). **IMPORTANT:** Validate input shape in the dispatcher BEFORE rendering the specialized component. This prevents dispatching malformed data to components that use React hooks (calling a hooks-using component as a plain function to check for null violates React Rules of Hooks). The component's own null-return for bad input is a last-resort safety net, not the primary fallback mechanism.

```typescript
// Validate input shape in dispatcher, then render via JSX (never call as plain function)
if (name === 'TaskCreate' && typeof input.subject === 'string') {
  return <TaskCreateBlock input={input} />
}
if (name === 'TaskUpdate' && typeof input.taskId === 'string') {
  return <TaskUpdateBlock input={input} />
}
if (name === 'TaskList') {
  return <TaskListBlock input={input} resultText={answerText ?? null} />
}
```

Note: `answerText` was already added to ToolCallBlockProps in Plan 01. For TaskList, the tool_result content serves as the result text showing the task summary. The same `answerText` prop (which carries the paired tool_result content) is reused — it's passed as `resultText` to TaskListBlock. TaskList needs no input validation because it takes no meaningful parameters (empty input is normal).

For TaskCreate and TaskUpdate, the tool_result is just a confirmation message (e.g., "Task #1 created successfully") which is not needed in the specialized display — the input fields contain all the meaningful data.

**2. Verify generic fallback is still last:**

Ensure the generic `summarizeToolInput` + generic card JSX remains as the final fallback after all specialized dispatch branches. The order should be:
1. AskUserQuestion -> AskUserQuestionBlock

3. TaskUpdate (with `typeof input.taskId === 'string'` guard) -> TaskUpdateBlock
4. TaskList -> TaskListBlock
5. Generic fallback (all other tools)

If input validation fails (e.g., TaskCreate without a string `subject`), the dispatch naturally falls through to the generic fallback — no special handling needed.
