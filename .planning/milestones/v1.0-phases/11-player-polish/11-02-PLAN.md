  <action>
    1. Modify `src/renderer/src/stores/playbackStore.ts`:
       - Add to PlaybackState interface:
         - `themeOverride: 'light' | 'dark' | null` (null = use file default)
         - `toggleTheme: () => void`
       - Initialize `themeOverride: null` in the store
       - Implement `toggleTheme`: reads the current effective theme (resolved from settings + override + system), then sets `themeOverride` to the opposite. Logic:
         ```
         const { presentation, themeOverride } = get()
         if (!presentation) return
         // Determine current effective theme
         const settingsTheme = presentation.settings.theme
         let current: 'light' | 'dark'
         if (themeOverride) {
           current = themeOverride
         } else if (settingsTheme === 'system') {
           current = document.documentElement.getAttribute('data-theme') === 'dark' ? 'dark' : 'light'
         } else {
           current = settingsTheme
         }
         set({ themeOverride: current === 'dark' ? 'light' : 'dark' })
         ```
       - In `loadPresentation`: reset `themeOverride: null` (ephemeral -- per user decision)
       - In `reset`: reset `themeOverride: null`

    2. Create `src/renderer/src/hooks/usePlayerTheme.ts`:
       - A hook that manages theme application when a presentation is active in the Player
       - Reads `presentation` and `themeOverride` from playbackStore
       - Reads system dark mode from appStore.isDarkMode
       - Resolves the effective theme:
         - If `themeOverride` is set, use it
         - Else if `presentation.settings.theme` is 'system', use appStore.isDarkMode to resolve
         - Else use `presentation.settings.theme` directly
       - On mount and when effective theme changes: set `document.documentElement.setAttribute('data-theme', effectiveTheme)`
       - Returns `{ effectiveTheme: 'light' | 'dark', isDark: boolean }` for consumers to read (e.g., SegmentedProgress toggle icon)
       - On cleanup (unmount): restore system theme by calling `window.electronAPI.getTheme()` and setting data-theme accordingly. This ensures leaving the Player restores the system-following behavior.

    3. Modify `src/renderer/src/hooks/useTheme.ts`:
       - Import `usePlaybackStore` from playbackStore
       - In the onThemeChange callback: before setting data-theme, check if a presentation is loaded (`usePlaybackStore.getState().presentation !== null`). If a presentation is loaded, do NOT update data-theme (the Player's usePlayerTheme manages it). Still update appStore.isDarkMode so the resolved 'system' value stays current.
       - Same guard in the initial getTheme().then() callback: only set data-theme when no presentation is loaded.
       - This prevents Pitfall 3: system theme changes overriding the Player's theme.
  </action>
