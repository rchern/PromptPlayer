Use `window.confirm()` for simplicity (or skip the prompt since nothing is lost due to auto-save -- per Research open question 2: "Your current presentation will remain available in the presentation list").
        2. Call `window.electronAPI.importPresentation()`
        3. If result is not null, call `importFromPromptPlay(result.presentation, result.sessions, result.filePath)`
        4. Switch to assembly view

   c. **Keyboard shortcut handlers**:
      - In a `useEffect` in the Builder component (or in the assembly view section):
        ```typescript
        useEffect(() => {
          const cleanupSave = window.electronAPI.onMenuSave(() => {
            // Only handle if we're in assembly view with an active presentation
            const active = usePresentationStore.getState().getActivePresentation()
            if (!active) return
            if (active.sourceFilePath) {
              // Overwrite existing file
              window.electronAPI.saveToPath(active.id, active.sourceFilePath)
            } else {
              // No source file -- trigger export (save-as)
              window.electronAPI.exportPresentation(active.id).then((result) => {
                if (!result.canceled) {
                  usePresentationStore.getState().setSourceFilePath(active.id, result.filePath)
                }
              })
            }
          })

          const cleanupSaveAs = window.electronAPI.onMenuSaveAs(() => {
            const active = usePresentationStore.getState().getActivePresentation()
            if (!active) return
            window.electronAPI.exportPresentation(active.id).then((result) => {
              if (!result.canceled) {
                usePresentationStore.getState().setSourceFilePath(active.id, result.filePath)
              }
            })
          })

          return () => {
            cleanupSave()
            cleanupSaveAs()
          }
        }, [])
        ```
      - NOTE: Using `usePresentationStore.getState()` in the callback (not a hook-level selector) because the callback is registered once and needs current state at invocation time.
      - Per Pitfall 4: Ctrl+S only overwrites when `sourceFilePath` is set. Otherwise it opens the save dialog (same as Ctrl+Shift+S).

   d. **Export success toast**:
      - Reuse the existing import toast pattern (useState + 3s auto-dismiss)
      - Show: "Exported to {filename}" or "Saved to {filename}"

IMPORTANT: When importing a .promptplay file, the sessions embedded in it get saved to app-local storage. This means the user can add more sessions from discovered JSONL files, modify the presentation, and re-export -- full editing capability per the locked decision.
