     a. **Tool Visibility** -- renders `<ToolVisibilityPanel />` with callbacks wired to store

   - Wire all callbacks to the presentation store actions (usePresentationStore)

   - Only render SettingsPanel when `activePresentationId` is set (need an active presentation to configure)

4. Wire live preview for the message preview area in the BROWSE view (the existing session detail panel):
   - Currently the detail panel renders `<MessageList messages={activeSession.messages} />` with no filtering
   - When in assembly view and a session is previewed, the messages should be filtered using `filterWithToolSettings` with the active presentation's `settings.toolVisibility`
   - Import `filterWithToolSettings` from utils/messageFiltering
   - In the assembly view, if there's an active session preview, wrap it:
     ```
     const filteredMessages = useMemo(() => {
       if (!activeSession || !activePresentation?.settings) return activeSession?.messages ?? []
       return filterWithToolSettings(activeSession.messages, activePresentation.settings.toolVisibility)
     }, [activeSession, activePresentation?.settings])
     ```
   - Pass `filteredMessages` to `<MessageList>` instead of raw messages
   - NOTE: The browse view (non-assembly) should continue showing unfiltered messages (no presentation settings apply in browse mode)

5. Apply scoped theme override to the message preview area ONLY (per Pitfall 7 / Research):
   - In the assembly view, when rendering the MessageList for preview, wrap it in a div with `data-theme={resolvedTheme}` where:
     ```
     const resolvedTheme = useMemo(() => {
       if (!activePresentation?.settings) return undefined
       const t = activePresentation.settings.theme
       if (t === 'system') return undefined  // inherit system theme
       return t  // 'light' or 'dark'
     }, [activePresentation?.settings?.theme])
     ```
   - Apply `data-theme` on the wrapper div around MessageList, NOT on documentElement
   - This ensures the Builder UI (settings panel, outline, etc.) stays in the system theme while the preview area reflects the presentation's configured theme

IMPORTANT: Do NOT modify the useTheme hook. The theme override is scoped entirely via the preview wrapper's data-theme attribute.
