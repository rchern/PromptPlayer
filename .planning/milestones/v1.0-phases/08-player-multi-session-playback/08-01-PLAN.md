Create `src/renderer/src/types/playback.ts` with:

1. **PlaybackStep discriminated union** with four variants:
   - `{ type: 'overview' }` — the title slide at index 0
   - `{ type: 'section-separator'; sectionId: string; sectionName: string; sessionCount: number; totalSteps: number }` — chapter title card
   - `{ type: 'session-separator'; sessionId: string; sessionName: string; sectionId: string; sectionName: string; stepCount: number; messageCount: number }` — session transition card
   - `{ type: 'navigation'; step: NavigationStep; sessionId: string; sectionId: string }` — wraps an existing NavigationStep with session/section context

2. **SectionProgressInfo** interface: `{ sectionId: string; sectionName: string; completed: number; total: number }` — used by the segmented progress bar (Plan 03).

Then create `src/renderer/src/stores/playbackStore.ts` with:

3. **`buildPlaybackSteps` function** (exported, pure):
   - Takes `(presentation: Presentation, sessionsMap: Map<string, StoredSession>)`.
   - Returns `PlaybackStep[]` starting with `{ type: 'overview' }`.
   - For each section: push a section-separator, then for each sessionRef: push a session-separator, then push wrapped navigation steps.
   - Use `filterWithToolSettings(session.messages, presentation.settings.toolVisibility)` to filter each session's messages (respects the Builder's per-tool visibility config).
   - Use `buildNavigationSteps(filteredMessages)` to produce NavigationStep[] per session.
   - Count section totalSteps as the sum of navigation steps (excluding separator cards) across all sessions in that section.
   - Skip missing sessions gracefully (session not in map → log console.warn, continue).

4. **`computeSectionProgress` function** (exported, pure):
   - Takes `(steps: PlaybackStep[], currentStepIndex: number, sections: PresentationSection[])`.
   - Returns `SectionProgressInfo[]` — for each section, count total navigation steps in that section and how many have been "completed" (their global index <= currentStepIndex).
   - Important: separator cards do NOT count in progress numerator or denominator (per Pitfall 2 from research).

Both functions are pure (no Zustand dependency) so they can be unit-tested independently.

<task type="auto">
  <name>Task 2: Create playback Zustand store with navigation and sidebar state</name>
  <files>src/renderer/src/stores/playbackStore.ts</files>
  <action>
In the same `playbackStore.ts` file (continuing from Task 1), add the Zustand store:

5. **PlaybackState interface** with:
   - `presentation: Presentation | null`
   - `sessions: Map<string, StoredSession>`
   - `steps: PlaybackStep[]`
   - `currentStepIndex: number`
   - `expandedSteps: Record<number, { user: boolean; assistant: boolean }>` — same pattern as navigationStore
   - `sidebarOpen: boolean`
   - `expandedSections: Set<string>` — section IDs expanded in sidebar tree

6. **Actions:**
   - `loadPresentation(presentation: Presentation, sessions: StoredSession[])`: Builds sessionsMap from array, calls buildPlaybackSteps, auto-expands all sections in sidebar, sets currentStepIndex to 0, clears expandedSteps, sets sidebarOpen to false.
   - `nextStep()`: Increment currentStepIndex by 1 if not at end. No special-casing for separator cards (per Pitfall 1 — separator cards are real steps).
   - `prevStep()`: Decrement by 1 if not at 0. Same rule — no special-casing.
   - `goToStep(index: number)`: Set currentStepIndex if in bounds.
   - `goToFirst()`: Set to 0.
   - `goToLast()`: Set to steps.length - 1.
   - `jumpToSection(sectionId: string)`: Find first step with `type === 'section-separator' && sectionId matches`, set currentStepIndex.
   - `jumpToSession(sessionId: string)`: Find first step with `type === 'session-separator' && sessionId matches`, set currentStepIndex.
   - `toggleExpand(stepIndex: number, role: 'user' | 'assistant')`: Same pattern as navigationStore.toggleExpand.
   - `toggleSidebar()`: Toggle sidebarOpen boolean.
   - `toggleSidebarSection(sectionId: string)`: Add/remove from expandedSections Set.
   - `reset()`: Clear all state to initial values.

7. Export as `usePlaybackStore` via `create<PlaybackState>()`.

Follow established Zustand patterns from navigationStore.ts and presentationStore.ts. Use `get()` for reading current state in actions, `set()` for mutations. Do NOT store computed/derived values (section progress, current section name) — those are computed via selectors in components.

Important implementation note: Zustand does not natively support `Set` for immutability detection. When mutating `expandedSections`, create a new `Set` instance (e.g., `const next = new Set(expandedSections)`) so Zustand detects the change.
