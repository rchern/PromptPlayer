**SegmentedProgress.tsx:**
Create a bottom-bar progress component with two parts: text and segmented bar.

Reads from playbackStore: `steps`, `currentStepIndex`, `presentation`.

Compute section progress using `computeSectionProgress(steps, currentStepIndex, presentation.sections)`.

Determine current section: find the section that the current step belongs to (by sectionId on the step). Compute local progress (which content step within this section) and global progress (currentStepIndex of total steps).

Layout (fixed to bottom of content area, spanning full width):
- **Text row** (right-aligned, above the bar):
  - If on overview: "Overview — 0 of {totalSteps} overall"
  - If on a separator or navigation step: "{sectionName} ({localStep}/{localTotal}) — {globalStep} of {globalTotal} overall"
  - Use monospace font with tabular-nums for stable width (same pattern as ProgressIndicator).
  - Font size: var(--text-sm), color: var(--color-text-muted).

- **Segmented bar** (full width, below the text):
  - Flex container with gap: 2px, height: 4px, border-radius: 2px.
  - One child div per section. Width proportional to section's content step count relative to total content steps: `(section.total / totalContentSteps) * 100%`.
  - Each segment has a background track (var(--color-bg-tertiary)) and a fill div (var(--color-accent)) with width proportional to section.completed / section.total.
  - Fill transition: `width 150ms ease` (same as existing ProgressIndicator).
  - Granularity: per-section segments (per research recommendation — 3-6 segments for visual clarity, not per-session which would be too granular).

Position: fixed at bottom of the content area (not the sidebar). Use `position: absolute; bottom: var(--space-3); left: var(--space-4); right: var(--space-4)` inside the content area div (which has `position: relative`).
