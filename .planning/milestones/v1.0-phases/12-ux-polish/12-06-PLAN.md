  <name>Task 1: Fix close/delete button event propagation in PresentationList</name>
  <files>src/renderer/src/components/builder/PresentationList.tsx</files>
  <action>
  In PresentationList.tsx, change both the close button and delete button from `onMouseDown` to `onClick`.

  **Close button (line 112):** Change `onMouseDown={(e) => handleClose(e, p.id, p.name)}` to `onClick={(e) => handleClose(e, p.id, p.name)}`.

  **Delete button (line 137):** Change `onMouseDown={(e) => handleDelete(e, p.id, p.name)}` to `onClick={(e) => handleDelete(e, p.id, p.name)}`.

  **Why this fixes the bug:** Both handlers already call `e.stopPropagation()`. When the child and parent both use `onClick`, `stopPropagation()` on the child's click event prevents it from reaching the parent's click handler. The current `onMouseDown` with `stopPropagation` only stops the mousedown from bubbling -- it does NOT prevent the browser from synthesizing a separate click event on the parent button, which immediately re-selects the tab after deselection.

  No other changes needed. The `handleClose` and `handleDelete` functions are already correct.
  </action>
