**Live preview reactivity (Builder.tsx):**

The current code at lines 251-257 uses `getActivePresentation()` (a function that calls `get().presentations.find(...)`) and then puts `activePresentation?.settings` as a useMemo dependency. The issue is that `getActivePresentation` returns the whole presentation object — and referential comparison of `activePresentation?.settings` may not reliably invalidate when deeply nested settings change because `persistPresentation` replaces the entire presentation object in the array.

Fix by using granular Zustand selectors instead of the computed getter:
1. Replace the `getActivePresentation()` call with direct store subscriptions. Add to the existing destructured store selectors:
   ```typescript
   const activePresentationFromStore = usePresentationStore((s) => {
     if (!s.activePresentationId) return null
     return s.presentations.find(p => p.id === s.activePresentationId) ?? null
   })
   ```
2. Use `activePresentationFromStore` everywhere that currently uses `activePresentation` (the `getActivePresentation()` result).
3. Update the `filteredMessages` useMemo dependency to use the specific settings reference:
   ```typescript
   }, [activeSession, activePresentationFromStore?.settings?.toolVisibility])
   ```
4. Update the `resolvedTheme` useMemo similarly — its dependency on `activePresentation?.settings?.theme` should already work since it's a primitive string, but verify.

This ensures Zustand's shallow comparison triggers re-renders when the presentation object in the array changes.
