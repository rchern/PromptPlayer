  <action>
Replace the scaffolded main process, preload, and renderer entry points with the PromptPlayer-specific implementation. Reference the code examples in 01-RESEARCH.md closely.

**src/main/index.ts** -- Electron main process:
- Create BrowserWindow with `frame: false`, `minWidth: 800`, `minHeight: 600`
- Default size: 80% of primary display work area (width and height)
- Restore saved window bounds from electron-store on subsequent launches (key: `windowBounds`)
- Save window bounds on close event
- `show: false` initially, show on `ready-to-show` (prevents white flash)
- `backgroundColor` set to `#ffffff`
- webPreferences: `contextIsolation: true`, `nodeIntegration: false`, `sandbox: false`, preload path pointing to preload/index.js
- Register IPC handlers: `window:minimize`, `window:maximize` (toggle), `window:close`, `window:isMaximized` (handle/invoke)
- Send `window:maximizeChanged` events to renderer on maximize/unmaximize BrowserWindow events
- Register theme IPC: `theme:get` returns `nativeTheme.shouldUseDarkColors`, listen to `nativeTheme.on('updated')` and send `theme:changed` to renderer
- Load renderer URL (dev) or file (production) per electron-vite convention
- Use `app.whenReady().then(createWindow)` pattern
- Handle `app.on('window-all-closed')` to quit on Windows

PITFALL (electron-store ESM): electron-store v10+ is ESM-only. If the main process config uses CommonJS, you may need dynamic `import()` or adjust the electron-vite config to output ESM for main. Check the scaffold's output format first. If electron-store import fails, use a simple JSON file read/write as fallback for window bounds persistence and note the issue.

**src/preload/index.ts** -- Context bridge:
- Expose `electronAPI` via `contextBridge.exposeInMainWorld` with:
  - `minimize()`, `maximize()`, `close()` -- fire-and-forget via `ipcRenderer.send()`
  - `isMaximized()` -- async via `ipcRenderer.invoke()`
  - `onMaximizeChange(callback)` -- returns cleanup function
  - `getTheme()` -- async via `ipcRenderer.invoke()`
  - `onThemeChange(callback)` -- returns cleanup function
- NEVER expose raw `ipcRenderer`

**src/renderer/src/types/electron.d.ts** -- TypeScript declarations:
- Define `ElectronAPI` interface matching preload exactly
- Declare `window.electronAPI` on global Window interface

**src/renderer/src/styles/theme.css** -- CSS custom properties:
- Copy the FULL theme.css from 01-RESEARCH.md code examples section (lines 398-494)
- Contains: teal accent colors, neutral palette, titlebar colors, shadows, spacing scale, typography (Inter + JetBrains Mono), border radii, titlebar height
- Includes complete `[data-theme="dark"]` overrides
- All color values use the teal palette per user's locked decision

**src/renderer/src/styles/global.css** -- Tailwind + base styles:
- Import Tailwind: `@import "tailwindcss";`
- Import theme: `@import "./theme.css";`
- Set `@theme` directive to integrate CSS variables with Tailwind if needed
- Base styles: `html, body { margin: 0; font-family: var(--font-sans); color: var(--color-text-primary); background: var(--color-bg-primary); }`, `*, *::before, *::after { box-sizing: border-box; }`
- Titlebar drag region styles: `.titlebar { -webkit-app-region: drag; }`, `.titlebar button, .titlebar a { -webkit-app-region: no-drag; }`

**src/renderer/src/stores/appStore.ts** -- Zustand store:
- State: `isDarkMode: boolean`, `recentFiles: RecentFile[]` (path, name, lastOpened)
- Actions: `setDarkMode`, `setRecentFiles`
- RecentFile interface: `{ path: string; name: string; lastOpened: number }`

**src/renderer/src/hooks/useTheme.ts** -- Theme detection hook:
- On mount: call `window.electronAPI.getTheme()`, set store and `data-theme` attribute
- Subscribe to `window.electronAPI.onThemeChange()`, update store and `data-theme`
- Clean up listener on unmount

**src/renderer/src/App.tsx** -- Router + app shell:
- Create `createHashRouter` with three routes: `/` (Home), `/builder` (Builder), `/player` (Player)
- For now, Home/Builder/Player can be inline placeholder components (simple div with text identifying the route)
- App component renders: a container div wrapping a Titlebar placeholder (simple div with "PromptPlayer" text and three placeholder buttons for min/max/close) and a `<RouterProvider>` for the main content area
- Call `useTheme()` hook in App component
- The Titlebar implementation will be a basic functional version here -- it will be refined in Plan 02

**src/renderer/src/main.tsx** -- React entry:
- Standard React 19 createRoot rendering App component
- Import global.css

**src/renderer/index.html** -- Update the renderer HTML:
- Ensure charset, viewport meta tags are present
- Title: "PromptPlayer"

Do NOT include fonts (Inter, JetBrains Mono) as bundled assets yet -- use the font stack fallbacks defined in theme.css (`system-ui`, etc.). Font bundling can happen in a later phase if needed. This avoids font file management complexity in the scaffold phase.
