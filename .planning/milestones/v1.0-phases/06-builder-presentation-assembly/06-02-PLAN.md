  <action>
    Create `src/renderer/src/stores/presentationStore.ts` as a new Zustand store following the pattern of `sessionStore.ts`.

    **State shape:**
    ```typescript
    interface PresentationState {
      // State
      presentations: Presentation[]
      activePresentationId: string | null
      isLoading: boolean

      // Computed helpers (not stored, derived)
      getActivePresentation: () => Presentation | null

      // CRUD
      loadPresentations: () => Promise<void>
      createPresentation: (sessions: SessionMetadata[]) => Promise<string>  // returns new presentation ID
      deletePresentation: (id: string) => Promise<void>
      setActivePresentation: (id: string | null) => void

      // Section manipulation
      mergeSections: (sectionIds: string[]) => void    // merges within active presentation
      removeSession: (sectionId: string, sessionId: string) => void  // removes session ref, removes section if empty

      // Add more sessions to active presentation
      addSessions: (sessions: SessionMetadata[]) => void

      // Naming
      renamePresentation: (id: string, name: string) => void
      renameSection: (sectionId: string, name: string) => void
      renameSessionRef: (sectionId: string, sessionId: string, name: string) => void
    }
    ```

    **Implementation details:**

    1. `loadPresentations`: Call `window.electronAPI.getPresentations()`, set presentations array. Called on store creation or app mount.

    2. `createPresentation(sessions)`: Use `createPresentationFromSessions(sessions)` from presentationUtils. Call `window.electronAPI.savePresentation(newPresentation)`. Add to local state. Set as active. Return the new ID.

    3. `deletePresentation(id)`: Call `window.electronAPI.deletePresentation(id)`. Remove from local state. If it was active, set activePresentationId to null.

    4. `getActivePresentation()`: Derive from `presentations.find(p => p.id === activePresentationId) ?? null`. Use `get()` inside the function.

    5. `mergeSections(sectionIds)`: Operates on active presentation. Must have 2+ section IDs. Collect all sessionRefs from matching sections, re-sort chronologically with `sortSessionRefsChronologically`. Create merged section using first section's ID and name. Replace in sections array: put merged at first section's index, filter out the rest. Update `updatedAt`. Persist via IPC. CRITICAL: Use immutable updates at every nesting level (spread presentation, map/filter sections array, spread merged section).

    6. `removeSession(sectionId, sessionId)`: Remove the SessionRef from the section's sessionRefs. If the section becomes empty after removal, remove the entire section. Update `updatedAt`. Persist.

    7. `addSessions(sessions)`: For each session, create a new PresentationSection (same as creation flow: one section per session, auto-named). Add sections to the active presentation's sections array. Re-sort ALL sections chronologically by earliest session sortKey. Persist.

    8. `renamePresentation/renameSection/renameSessionRef`: Immutable update at the correct nesting level. Update `updatedAt`. Persist.

    **Persistence helper**: Create a private `persistPresentation(presentation: Presentation)` helper that calls `window.electronAPI.savePresentation(presentation)` and updates the local presentations array immutably. All mutating operations should call this after computing the new presentation state.

    Import types from '../types/presentation' and '../types/pipeline'. Import utilities from '../utils/presentationUtils'.
  </action>

  </verify>

    **Step 1: Add selection state to sessionStore.ts**

    Add to the SessionState interface:
    ```typescript
    // Selection state (for presentation creation flow)
    selectedSessionIds: Set<string>
    isSelecting: boolean

    // Selection actions
    toggleSessionSelection: (sessionId: string) => void
    clearSelection: () => void
    setSelecting: (selecting: boolean) => void
    ```

    Initial state: `selectedSessionIds: new Set(), isSelecting: false`

    `toggleSessionSelection`: Create new Set from existing, add or delete sessionId, set new Set.
    `clearSelection`: Set `selectedSessionIds` to new empty Set, `isSelecting` to false.
    `setSelecting`: Set `isSelecting`. If setting to false, also clear `selectedSessionIds` (prevents stale selections per Research pitfall 5).

    **Step 2: Add selectable mode to SessionCard.tsx**

    Extend SessionCardProps with optional props:
    ```typescript
    selectable?: boolean
    isSelected?: boolean
    onToggleSelect?: (sessionId: string) => void
    ```

    In the render, when `selectable` is true, add a checkbox as the first element inside the card button (before the existing top row):
    ```tsx
    {selectable && (
      <div
        className="flex items-center"
        style={{ position: 'absolute', top: 'var(--space-3)', right: 'var(--space-3)' }}
      >
        <input
          type="checkbox"
          checked={isSelected}
          onChange={(e) => {
            e.stopPropagation()
            onToggleSelect?.(session.sessionId)
          }}
          onClick={(e) => e.stopPropagation()}
          style={{ accentColor: 'var(--color-accent)', width: '16px', height: '16px', cursor: 'pointer' }}
        />
      </div>
    )}
    ```

    Add `position: 'relative'` to the card's button style when selectable is true (to position the checkbox absolutely). Add right padding when selectable to avoid text overlapping the checkbox (`paddingRight: selectable ? 'var(--space-8)' : 'var(--space-4)'`).

    When selectable, clicking the card should toggle selection instead of opening preview. Modify the onClick:
    ```tsx
    onClick={() => {
      if (selectable) {
        onToggleSelect?.(session.sessionId)
      } else {
        onSelect(session)
      }
    }}
    ```

    When selected, the card should show a selected visual state (accent border, subtle accent bg) similar to the active state.

    ```tsx
      showProject={/* existing logic */}

      onToggleSelect={onToggleSelect}
    />
    ```
