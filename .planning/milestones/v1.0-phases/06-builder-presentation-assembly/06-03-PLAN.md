  <action>
    **Component 1: InlineEdit.tsx** (~30 lines of logic)

    Reusable click-to-edit component. Props: `{ value: string, onSave: (newValue: string) => void, className?: string, inputClassName?: string }`.

    Behavior:
    - Displays `value` in a `<span>` with `cursor: text`
    - On hover, shows dashed underline hint (borderBottom: '1px dashed var(--color-text-muted)')
    - On click, switches to `<input>` pre-filled with current value, auto-focused, text selected
    - Enter or blur: save trimmed value if non-empty and different from original; revert if empty/unchanged
    - Escape: revert without saving
    - Sync external value changes when not editing (useEffect on value when !isEditing)

    PITFALL (from Research): Do NOT use contentEditable. Use a simple input toggle pattern.

    **Component 2: SessionEntry.tsx** (React.memo for performance)

    Props: `{ sessionRef: SessionRef, sectionId: string, onRename: (sectionId: string, sessionId: string, name: string) => void, onRemove: (sectionId: string, sessionId: string) => void }`

    Displays one session row within a section:
    - InlineEdit for the display name (calls onRename on save)
    - Compact metadata: sortKey formatted as short date/time (use `new Date(sortKey).toLocaleString()` if sortKey exists)
    - Remove button (X icon from lucide-react, small, muted, appears on hover): calls onRemove
    - Use `onMouseDown` with `e.preventDefault()` on the remove button to prevent InlineEdit blur conflicts (Research pitfall 3)

    Wrap with React.memo for performance with 30+ sessions (Research pitfall 6).

    **Component 3: SectionHeader.tsx** (React.memo for performance)

    Props: `{ section: PresentationSection, isSelected: boolean, onToggleSelect: (sectionId: string) => void, onRename: (sectionId: string, name: string) => void }`

    Displays one section header:
    - Checkbox for merge selection (checked = isSelected, toggle on change)
    - Section icon (lucide-react `Layers` or `FolderOpen`)
    - InlineEdit for section name (calls onRename on save)
    - Session count badge: "{n} session(s)" in muted text
    - Visual distinction: slightly larger text than SessionEntry, accent color for section names, subtle background

    Wrap with React.memo.

    **Component 4: PresentationOutline.tsx**

    The main hierarchical view showing all sections and their sessions for the active presentation.

    Connects to `usePresentationStore` to get the active presentation and call actions.

    Layout:
    - If no active presentation: show empty state "No presentation selected"
    - Presentation name at top as InlineEdit (large, bold) - calls renamePresentation
    - "Merge Selected" button: disabled when fewer than 2 sections are selected. On click, calls mergeSections with selected section IDs. Clears selection after merge.
    - Section list: map over presentation.sections, render SectionHeader + SessionEntry list for each section
    - Uses local state `selectedSectionIds: Set<string>` for merge selection (not global store -- only needed in this component)

    Wire the callbacks:
    - SectionHeader.onRename -> presentationStore.renameSection
    - SectionHeader.onToggleSelect -> local selectedSectionIds state
    - SessionEntry.onRename -> presentationStore.renameSessionRef
    - SessionEntry.onRemove -> presentationStore.removeSession

    **Component 5: PresentationList.tsx**

    Compact list/switcher for managing multiple presentations.

    Connects to `usePresentationStore`.

    Layout:
    - Compact horizontal or vertical list above the outline
    - Each presentation: name (truncated), date created, active indicator (accent border/bg)
    - Click to switch active presentation (calls setActivePresentation)
    - Delete button (Trash2 icon, appears on hover) with confirmation (simple window.confirm)
    - "New Presentation" button that enters selection mode (sets sessionStore.setSelecting(true) and switches Builder to browse view)

    Keep this compact -- it's a utility bar, not a primary UI element. Think tabs or a small sidebar list.

  <action>
    Refactor Builder.tsx to support two views: `'browse'` (existing session browser) and `'assembly'` (new presentation editor).

    **State additions:**
    ```typescript
    type BuilderView = 'browse' | 'assembly'
    const [view, setView] = useState<BuilderView>('browse')
    ```

    Connect to both stores:
    ```typescript
    const { selectedSessionIds, isSelecting, setSelecting, clearSelection, toggleSessionSelection } = useSessionStore()
    const { loadPresentations, createPresentation, presentations, activePresentationId, setActivePresentation } = usePresentationStore()
    ```

    Load presentations on mount (alongside existing discover):
    ```typescript
    useEffect(() => {
      discover()
      loadPresentations()
    }, [])
    ```

    **Browse view modifications:**

    The existing browse view gets two additions:

    1. A "Select for Presentation" toggle button in the header (next to Import/Refresh/Browse). When clicked, enters selection mode (isSelecting=true). When in selection mode, changes to "Cancel Selection" button.

    2. When `isSelecting` is true:
       - Pass `selectable={true}`, `selectedSessionIds`, and `onToggleSelect={toggleSessionSelection}` to SessionList
       - Show a floating action bar at the bottom of the session list area: "{N} selected" count + "Create Presentation" button (accent-colored, disabled when 0 selected)
       - "Create Presentation" button: calls `createPresentation(selectedSessions)` where selectedSessions is the discovered sessions filtered by selectedSessionIds. Then clears selection, exits selection mode, switches to assembly view.

    3. If the user arrives at browse view because PresentationList's "New Presentation" button was clicked (isSelecting is already true), show the selection mode immediately.

    **Assembly view:**

    When `view === 'assembly'`:
    ```tsx
    <div className="flex" style={{ height: '100%', gap: 'var(--space-4)' }}>
      {/* Left panel: compact session library for adding more sessions */}
      <div className="flex flex-col" style={{ flex: '0 0 40%', overflow: 'hidden' }}>
        {/* Header with "Back to Browser" button */}
        <div className="flex items-center justify-between" style={{ marginBottom: 'var(--space-3)', flexShrink: 0 }}>
          <h3 style={{ fontSize: 'var(--text-lg)', fontWeight: 600, color: 'var(--color-text-primary)' }}>
            Add Sessions
          </h3>
          <button onClick={() => { setView('browse'); clearSelection() }}>
            Back to Browser
          </button>
        </div>
        {/* Search/filter bar (compact) */}
        <SearchFilterBar ... />
        {/* Session list in selectable mode for adding */}
        <div style={{ flex: 1, overflowY: 'auto', minHeight: 0 }}>
          <SessionList
            sessions={filteredSessions}
            onSelectSession={handleSelectSession}
            isLoading={isDiscovering}
            viewMode={viewMode}
            activeSessionId={null}
            selectable={true}
            selectedSessionIds={selectedSessionIds}
            onToggleSelect={toggleSessionSelection}
          />
        </div>
        {/* "Add to Presentation" button when sessions are selected */}
        {selectedSessionIds.size > 0 && (
          <button onClick={handleAddSessions}>
            Add {selectedSessionIds.size} Session(s)
          </button>
        )}
      </div>

      {/* Right panel: presentation outline */}
      <div className="flex flex-col" style={{ flex: '1 1 60%', overflow: 'hidden' }}>
        <PresentationList />
        <PresentationOutline />
      </div>
    </div>
    ```

    `handleAddSessions`: Find the SessionMetadata objects for selectedSessionIds from discoveredSessions. Call `presentationStore.addSessions(selectedMetadata)`. Clear selection.

    **Styling for assembly view:**
    - Use the existing theme tokens (--color-bg-secondary, --color-border, etc.)
    - Left panel has a subtle border-right or separator
    - Right panel is the primary focus area
    - "Add to Presentation" button uses accent styling when sessions are selected
    - "Back to Browser" button is understated (secondary styling)

    **Auto-navigate to assembly view:** If `presentations.length > 0 && activePresentationId` and user hasn't explicitly gone to browse, consider showing assembly. But default to browse on fresh app start per the creation flow (select then create). This is discretionary -- keep it simple: always start in browse view, user navigates to assembly via creation or PresentationList.

    Style all buttons consistently with the existing Builder button patterns (the Import/Refresh/Browse buttons at the top).
